Lab 2 – Front-end Web Development

In Lab 2, I built a front-end web page from scratch using HTML, CSS, and JavaScript, and integrated it with external APIs. The lab covered two main areas:

Task 1: Basic HTML & JavaScript
  • Built a base HTML page with a headshot, form, and multiple JavaScript features.
  • Used inline JavaScript to display the current date/time on click and log keypresses to the console.
  • Implemented a digital clock via setInterval inside a <script> tag.
  • Added a show/hide email toggle using an external JavaScript file.
  • Rendered an analog clock using an external analog.js file with canvas-based hand rotation.
Task 2: Ajax, CSS, jQuery, and Web API Integration
  • Applied all three CSS types: inline, internal <style> block, and external via Bootstrap CDN.
  • Used XMLHttpRequest to send an Ajax GET request to echo.php, then inspected the 200 OK response in the browser network tab.
  • Re-implemented both Ajax calls using jQuery ($.ajax()) with two separate buttons, drastically reducing code length.
  • Integrated the JokeAPI (v2.jokeapi.dev/joke/Programming) using jQuery Ajax to display a random programming joke on page load.
  • Integrated the Agify API using the fetch() method with chained .then() handlers — users type a name and get a predicted age back in real time.

Hackathon 1 – XSS Attacks & Defenses

Hackathon 1 was a full-day exercise in both exploiting and defending against Cross-Site Scripting (XSS) vulnerabilities on a live target application hosted at waph-hackathon.eastus.cloudapp.azure.com/xss.

Task 1: XSS Attack Simulation (Levels 0–6)

Each level of echo.php had a different server-side filtering strategy. I injected payloads to trigger alert() displaying my name, and guessed the underlying PHP source code for levels 2–6:

  • Level 0–1: No filtering — basic <script>alert()</script> payload worked directly.
  • Level 2: Server reflected input with no sanitization — same direct payload.
  • Level 3: Server stripped <script> tags — bypassed with case variation or alternative tags (e.g., <img onerror=...>).
  • Level 4: Keyword blocklist on "script" — bypassed using an event handler attribute that avoids the blocked string.
  • Level 5: Blocklist also covering "alert" — bypassed by encoding or alternative JS execution methods.
  • Level 6: Proper htmlspecialchars() encoding — this level was not bypassable; the defense worked correctly.
